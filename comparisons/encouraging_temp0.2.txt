**Question 2 of 9:**

Which of the following is a valid way to write a multi-line comment in Python?

**A)** `/* This is a comment */`  
**B)** `# This is a comment`  
**C)** `// This is a comment`  
**D)** A triple-quoted string: `"""This is a comment"""`  

Now that you can see the question, would you like to answer it, or would you like a hint first?


Not quite! Let me explain why.

**The correct answer is D) A triple-quoted string: `"""This is a comment"""`**

Here's the breakdown:

- **Option A** (`/* */`) is C-style syntax — doesn't work in Python
- **Option B** (`#`) is correct for **single-line** comments, but the question asks specifically for a **multi-line** comment
- **Option C** (`//`) is JavaScript/C-style syntax — doesn't work in Python
- **Option D** is the Python convention! Triple-quoted strings (`"""..."""` or `'''...'''`) are used as multi-line comments (also called docstrings)

**Key takeaway:** While you *can* use multiple `#` symbols on separate lines for multi-line comments, the idiomatic Python way is to use triple quotes.
