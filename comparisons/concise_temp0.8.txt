**Question 1 of 9:**

What happens when you run the following Python code?
```python
x = 10
y = 20
print(x + y)
```

A) Prints 30  
B) Prints 10 20  
C) Raises an IndentationError  
D) Raises a SyntaxError about the variable name


💡 Hint: The code appears simple at first glance, but pay close attention to the indentation of each line.


Incorrect. The indentation of `y = 20` causes an IndentationError because it's unexpectedly indented without belonging to a required block.

**Question 2 of 9:**

Which of the following is a valid way to write a multi-line comment in Python?

A) `/* This is a comment */`  
B) `# This is a comment`  
C) `// This is a comment`  
D) A triple-quoted string: `"""This is a comment"""`


💡 Hint: Python doesn't use C-style or JavaScript-style comment syntax; think about what Python uses for docstrings.


Incorrect. While `#` works for single-line comments, the question asks for multi-line comments, which use triple-quoted strings.

**Question 3 of 9:**

What is the output of the following code?
```python
name = 'Alice'
print('Hello,', name, end='!\n')
```

A) Hello, Alice!  
B) Hello,Alice!  
C) Hello, Alice!\n  
D) Hello, Alice !


💡 Hint: The `end` parameter replaces the default newline, and `print()` separates arguments with spaces by default.


Correct. The `print()` function adds a space between arguments and uses the `end` parameter to replace the default newline.

**Question 4 of 9:**

Given the code below, what does `z` hold after execution?
```python
x = 5
y = x
x = 10
z = y
```

A) 5  
B) 10  
C) 15  
D) Raises a NameError


💡 Hint: When you assign `y = x`, `y` gets the value at that moment; changing `x` later doesn't affect `y`.


Incorrect. When `y = x` executes, `y` gets the value 5; later reassigning `x = 10` doesn't change `y`, so `z` holds 5.
