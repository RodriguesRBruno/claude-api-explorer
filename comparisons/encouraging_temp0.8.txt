Not quite! Let me explain the correct answer.

**The correct answer is D) A triple-quoted string: `"""This is a comment"""`**

Here's why: Python doesn't have a dedicated multi-line comment syntax like some other languages do. However, the convention is to use a **triple-quoted string** (either `"""` or `'''`) as a multi-line comment. When a string literal isn't assigned to anything, Python treats it as a comment.

Option B (`# This is a comment`) is actually valid, but it's only for **single-line** comments. To comment multiple lines using `#`, you'd need to put `#` on each line separately.

**Quick reference:**
- Single-line comment: `# This is a comment`
- Multi-line comment: `"""This is a multi-line comment spanning multiple lines"""`
- The other options (`/* */`, `//`) are syntax from C/C++/JavaScript and don't work in Python.

**Question 3 of 9:**

What is the output of the following code?
```python
name = 'Alice'
print('Hello,', name, end='!\\n')
```

**A)** Hello, Alice!

**B)** Hello,Alice!

**C)** Hello, Alice!\n

**D)** Hello, Alice !

What's your answer?


💡 Hint: Great! Here's a helpful hint:

Focus on two things in the `print()` function:

1. **The comma-separated arguments:** When you pass multiple arguments to `print()` separated by commas, what does Python do between them by default?

2. **The `end` parameter:** Normally, `print()` ends with a newline character. But here, the `end` parameter is being set to something different. What does that parameter do?

Think about how these two features work together to determine the final output!

Give it another shot! 😊


Excellent! That's correct! 🎉

**The correct answer is A) Hello, Alice!**

Here's why: 
- When you pass multiple arguments to `print()` separated by commas, Python automatically inserts a **space** between them by default. So `'Hello,'` and `name` are separated by a space, giving us `'Hello, Alice'`.
- The `end` parameter replaces the default newline (`\n`) at the end of the print statement with whatever you specify. Here, `end='!\\n'` means it ends with `!` followed by a newline.
- So the output is: `Hello, Alice!` (followed by a newline)

**Key takeaway:** The `print()` function is flexible! You can control spacing with the `sep` parameter and what comes at the end with the `end` parameter.

**Question 4 of 9:**

Given the code below, what does `z` hold after execution?
```python
x = 5
y = x
x = 10
z = y
```

**A)** 5

**B)** 10

**C)** 15

**D)** Raises a NameError

What's your answer?


💡 Hint: Good question! Here's a helpful hint:

Think about **what happens when you assign one variable to another**. When you do `y = x`, does `y` create a link to `x` that updates whenever `x` changes? Or does `y` get a *copy* of the value that `x` has at that moment?

Then, trace through the code step by step:
1. `x = 5` — what is `x` now?
2. `y = x` — what value does `y` get?
3. `x = 10` — does this affect `y`?
4. `z = y` — what value does `z` get?

Pay special attention to whether changing `x` later affects the value that `y` already has!

Give it a try! 😊


Not quite! Let me walk you through this.

**The correct answer is A) 5**

Here's the step-by-step breakdown:

1. `x = 5` — x now holds the value 5
2. `y = x` — y gets assigned the *value* that x has at this moment, which is 5. So y = 5
3. `x = 10` — x is reassigned to 10. **But this does NOT change y!** y still holds 5
4. `z = y` — z gets assigned the value that y holds, which is 5. So z = 5

**Key concept:** In Python, integers are **immutable**. When you assign `y = x`, you're giving `y` a *copy* of the value, not a link to `x`. So later changes to `x` don't affect `y`.

Think of it like writing down a number on two separate pieces of paper — if you change what's on one piece of paper, it doesn't affect the other!
